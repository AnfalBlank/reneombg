1. Login & Navigasi

1.1 Login ke Sistem

Untuk mengakses sistem ERP MBG, Anda memerlukan akun yang telah didaftarkan oleh Super Admin.

📸 Screenshot: Halaman Login ERP MBG
Gambar 1.1 — Halaman login dengan form email dan password
  1. Buka browser dan akses URL https://rmb.manggalautama.web.id
  2. Masukkan Email yang telah didaftarkan
  3. Masukkan Password Anda
  4. Klik tombol Masuk
  5. Sistem akan mengarahkan ke Dashboard sesuai role Anda
💡 Tip: Jika lupa password, hubungi Super Admin untuk melakukan reset password melalui menu Pengaturan → Pengguna.

1.2 Navigasi Sidebar

Setelah login, Anda akan melihat sidebar navigasi di sisi kiri layar. Menu yang tampil berbeda sesuai role pengguna.

📸 Screenshot: Sidebar navigasi dengan menu utama
Gambar 1.2 — Sidebar navigasi menampilkan menu sesuai role pengguna
  • Sidebar kiri: Menu utama (berbeda per role)
  • Header atas: Logo, notifikasi 🔔, chat 💬, profil
  • Klik menu untuk membuka halaman
  • Submenu: Klik grup menu untuk expand/collapse

1.3 Logout

Klik ikon Logout (↪) di bagian bawah sidebar untuk keluar dari sistem.

⚠️ Perhatian: Selalu logout setelah selesai menggunakan sistem, terutama jika menggunakan komputer bersama.

3. Master Data

Master Data adalah data dasar yang digunakan di seluruh modul sistem. Pastikan data master terisi lengkap sebelum memulai transaksi.

3.1 Item / SKU

Kelola daftar item/bahan yang digunakan dalam operasional. Setiap item memiliki kode SKU unik yang di-generate otomatis.

📸 Screenshot: Halaman Master Data Item / SKU
Gambar 3.1 — Daftar item dengan tombol tambah dan edit

Menambah Item Baru

  1. Buka Master Data → Item / SKU
  2. Klik tombol + Tambah Item
  3. Isi form: Nama, Kategori, Satuan (UOM), Min. Stok
  4. SKU otomatis di-generate (ITM-0001, dst)
  5. Klik Simpan

Mengedit Item

  1. Klik tombol Edit pada baris item yang ingin diubah
  2. Ubah data yang diperlukan
  3. Klik Simpan

3.2 Vendor

Kelola data vendor/supplier yang menyediakan bahan baku.

Menambah Vendor

  1. Buka Master Data → Vendor
  2. Klik + Tambah Vendor
  3. Isi form: Nama, Kontak, Telepon, Email, Kategori
  4. Klik Simpan
💡 Tip: Kategori vendor membantu memfilter vendor saat membuat Purchase Order.

3.3 Dapur / Unit

Kelola data dapur/unit operasional yang menjadi tujuan distribusi bahan.

Menambah Dapur

  1. Buka Master Data → Dapur / Unit
  2. Klik + Tambah Dapur
  3. Isi form: Nama, Lokasi, PIC (Person in Charge)
  4. Klik Simpan

3. Master Data (lanjutan)

3.4 Resep / BOM (Bill of Materials)

Kelola resep menu beserta komposisi bahan. Resep digunakan untuk menghitung kebutuhan bahan saat membuat Internal Request.

📸 Screenshot: Halaman Resep / BOM dengan daftar resep dan detail bahan
Gambar 3.2 — Daftar resep dengan scaling simulator

Membuat Resep Manual

  1. Buka Master Data → Resep / BOM
  2. Klik + Buat Resep
  3. Isi nama menu dan default porsi
  4. Tambahkan bahan: pilih item, qty, satuan
  5. Klik Simpan

Upload dari Template Excel

  1. Klik Upload Template
  2. Pilih file Excel (format SPPG)
  3. Sistem auto-detect menu, bahan, dan porsi
  4. Item yang belum ada di master data otomatis dibuat
  5. Review hasil parsing dan klik Simpan

Scaling Simulator

  1. Buka detail resep yang sudah ada
  2. Masukkan target porsi yang diinginkan
  3. Sistem otomatis menghitung kebutuhan bahan sesuai proporsi
💡 Tip: Scaling simulator sangat berguna untuk menghitung kebutuhan bahan saat ada event atau pesanan dalam jumlah besar.

4. Pembelian

Modul pembelian mengelola proses pengadaan barang dari vendor, mulai dari pembuatan PO hingga penerimaan barang di gudang.

4.1 Purchase Order (PO)

📸 Screenshot: Form pembuatan Purchase Order
Gambar 4.1 — Form PO dengan pilihan vendor, gudang, dan daftar item

Membuat PO

  1. Buka Pembelian → Purchase Order
  2. Klik + Buat PO
  3. Pilih Vendor dan Gudang tujuan
  4. Tambahkan item: pilih item dari dropdown, masukkan Qty dan Harga Satuan (otomatis format Rp)
  5. Klik Simpan — status PO menjadi Draft

Mengajukan Approval

  1. Pada PO dengan status Draft, klik tombol Ajukan
  2. Status berubah menjadi Pending Approval
  3. Admin/Super Admin akan menerima notifikasi untuk review

Approve PO

  1. Buka PO dengan status Pending
  2. Review detail PO (vendor, item, harga)
  3. Klik Approve — status berubah menjadi Open (siap diterima)
⚠️ Perhatian: PO yang sudah di-approve tidak dapat diedit. Pastikan semua data sudah benar sebelum approve.

4.2 Goods Receipt (Penerimaan Barang)

Menerima Barang dari Vendor

  1. Buka Pembelian → Goods Receipt
  2. Pilih PO yang statusnya Open
  3. Klik Terima Barang
  4. Input qty aktual yang diterima per item
  5. Klik Konfirmasi
  6. Stok gudang otomatis bertambah
  7. Record pembayaran vendor otomatis dibuat di modul Arus Kas
💡 Tip: Setelah Goods Receipt, tagihan vendor otomatis muncul di menu Pembayaran Vendor untuk proses pembayaran.
Manual Book ERP MBG v1.1.0 — PT. Manggala Utama Indonesia

5.1 Internal Request (IR)

Internal Request adalah permintaan bahan dari dapur ke gudang pusat.

📸 Screenshot: Form Internal Request dengan info anggaran dan daftar item
Gambar 5.1 — Form IR menampilkan budget warning dan daftar item yang diminta

Membuat IR Manual

  1. Buka Supply Chain → Internal Request
  2. Klik + Buat IR
  3. Pilih Dapur Peminta (admin dapur: otomatis terisi)
  4. Pilih Gudang Sumber
  5. Perhatikan info anggaran dapur (jika ada budget aktif)
  6. Tambahkan item: pilih item, masukkan qty
  7. Klik Simpan

Membuat IR dari Resep/BOM

  1. Di form IR, buka bagian Load dari Resep
  2. Pilih resep dan masukkan target porsi
  3. Klik Load — item otomatis terisi sesuai resep
  4. Sesuaikan qty jika perlu
  5. Klik Simpan

Upload IR dari Excel

  1. Di form IR, klik Upload Excel
  2. Pilih file template SPPG (.xlsx)
  3. Sistem auto-detect: dapur, menu, item, qty
  4. Item baru otomatis dibuat di master data
  5. Menu baru otomatis dibuat di BOM
  6. Review hasil parsing dan klik Simpan
⚠️ Budget Warning: Jika dapur memiliki anggaran aktif, sistem akan menampilkan sisa budget dan peringatan jika IR melebihi anggaran.

Approve IR

  1. Buka IR dengan status Pending
  2. Review item dan qty yang diminta
  3. Klik Approve
  4. Delivery Order otomatis dibuat (status Draft)
  5. Notifikasi dikirim ke peminta (web + Telegram)
Manual Book ERP MBG v1.1.0 — PT. Manggala Utama Indonesia

5. Supply Chain (lanjutan)

5.2 Delivery Order (DO)

Delivery Order dibuat otomatis setelah IR di-approve. Admin gudang melengkapi harga dan mengirim barang.

📸 Screenshot: Halaman Delivery Order dengan daftar DO dan status
Gambar 5.2 — Daftar DO dengan tombol edit harga dan kirim

Melengkapi DO (Input Harga)

  1. Buka Supply Chain → Delivery Order
  2. Pilih DO dengan status Draft
  3. Klik Edit → input harga jual per item
  4. Klik Simpan

Konfirmasi Pengiriman

  1. Klik Kirim pada DO yang sudah lengkap
  2. Stok gudang berkurang sesuai qty
  3. Status berubah menjadi Terkirim
  4. Notifikasi dikirim ke admin dapur (web + Telegram)
  5. Surat jalan bisa di-print (PDF)

5.3 Kitchen Receiving (KR)

Kitchen Receiving adalah proses penerimaan barang di dapur. Admin dapur memverifikasi qty aktual yang diterima.

📸 Screenshot: Popup tabel Kitchen Receiving dengan kolom dikirim, diterima, selisih, dan alasan
Gambar 5.3 — Tabel KR menampilkan detail item dengan input qty aktual dan alasan penolakan

Menerima Barang di Dapur

  1. Buka Supply Chain → Kitchen Receiving
  2. Di bagian Menunggu Penerimaan, klik Terima Barang
  3. Popup tabel muncul dengan kolom: Dikirim, Diterima, Selisih, Alasan
  4. Input qty aktual yang diterima (default = qty dikirim)
  5. Jika ada item ditolak, isi kolom Alasan
  6. Isi catatan penerimaan (opsional)
  7. Lihat summary: Total Dikirim, Diterima, Ditolak
  8. Klik Konfirmasi Penerimaan
🔴 Penting: Setelah konfirmasi KR, sistem otomatis: (1) menambah stok dapur, (2) membuat invoice, (3) mencatat item ditolak ke Pengembalian, (4) membuat jurnal distribusi, (5) mengirim notifikasi Telegram.

5.4 Pemakaian Bahan

Catat penggunaan bahan di dapur untuk mengurangi stok dapur.

  1. Buka Supply Chain → Pemakaian Bahan
  2. Pilih dapur
  3. Pilih resep (opsional) atau input manual
  4. Masukkan item dan qty yang dipakai
  5. Klik Simpan — stok dapur berkurang

6. Inventori

Modul Inventori mengelola stok barang di gudang dan dapur, termasuk stock opname dan pengembalian barang.

6.1 Stok Gudang

📸 Screenshot: Halaman Stok Gudang dengan tabel stok per lokasi
Gambar 6.1 — Daftar stok gudang dengan indikator stok minimum
  1. Buka Inventori → Stok Gudang
  2. Lihat stok per gudang dan per dapur
  3. Item dengan stok di bawah minimum ditandai merah
  4. Gunakan filter untuk melihat stok per lokasi tertentu
💡 Tip: Stok otomatis terupdate saat terjadi Goods Receipt (masuk), Delivery Order (keluar), Kitchen Receiving (masuk dapur), dan Pemakaian Bahan (keluar dapur).

6.2 Stock Opname

Stock Opname digunakan untuk mencocokkan stok fisik dengan stok di sistem.

  1. Buka Inventori → Stock Opname
  2. Klik + Buat Stock Opname
  3. Pilih gudang yang akan di-opname
  4. Input qty fisik per item
  5. Selisih otomatis terhitung (fisik − sistem)
  6. Klik Simpan
  7. Hasil opname bisa di-view dan download PDF

6.3 Pengembalian Barang

Item yang ditolak saat Kitchen Receiving otomatis masuk ke daftar pengembalian.

  1. Buka Inventori → Pengembalian
  2. Lihat daftar item yang Pending Return (dari KR partial)
  3. Klik Detail untuk lihat info lengkap (alasan penolakan, qty)
  4. Klik Approve untuk mengembalikan item ke gudang
  5. Stok gudang bertambah kembali sesuai qty yang dikembalikan

7. Keuangan (Arus Kas)

Modul Keuangan mengelola seluruh arus kas operasional: pembayaran vendor, invoice dapur, anggaran, dan pengeluaran.

7.1 Pembayaran Vendor

  1. Buka Arus Kas → Pembayaran Vendor
  2. Klik Sync untuk generate record dari Goods Receipt
  3. Daftar tagihan vendor muncul (status: Belum Bayar)
  4. Klik Upload Bukti → pilih file (JPG/PDF) → upload
  5. Status berubah menjadi Pending
  6. Finance klik Approve → status menjadi Lunas

7.2 Invoice Dapur

Invoice otomatis terbit setelah Kitchen Receiving dikonfirmasi. Berisi detail item yang diterima beserta harga.

📸 Screenshot: Halaman Invoice Dapur dengan filter, rekap, dan daftar invoice
Gambar 7.1 — Halaman invoice dengan summary tagihan dan status pembayaran

Filter & Rekap

  • Filter: search, dapur, bulan, tahun, status
  • Rekap per Dapur: tabel ringkasan (jika lebih dari 1 dapur)
  • Summary: Total Tagihan, Lunas, Belum Bayar

Membayar Invoice

  1. Klik Bayar pada invoice dengan status Belum Bayar
  2. Isi tanggal bayar dan metode pembayaran
  3. Upload bukti pembayaran (JPG/PDF) — wajib
  4. Klik Upload & Ajukan
  5. Status berubah menjadi Pending
📸 Screenshot: Modal pembayaran invoice dengan upload bukti
Gambar 7.2 — Modal pembayaran dengan form tanggal, metode, dan upload bukti

Approve Pembayaran

  1. Klik Approve pada invoice dengan status Pending
  2. Konfirmasi → status menjadi Lunas

Cetak Invoice

  • PDF per invoice: klik tombol PDF pada baris invoice
  • Rekap periode: klik Cetak Rekap di header halaman
💡 Tip: Admin Dapur hanya melihat invoice dapurnya sendiri dan tidak bisa melakukan approve pembayaran.

7.3 Anggaran Dapur

Kelola anggaran operasional per dapur per periode. Sistem memantau penggunaan anggaran secara real-time.

📸 Screenshot: Halaman Anggaran Dapur dengan progress bar dan badge status
Gambar 7.3 — Daftar anggaran dengan progress bar hijau/kuning/merah

Membuat Anggaran

  1. Buka Arus Kas → Anggaran Dapur
  2. Klik + Buat Anggaran
  3. Pilih dapur tujuan
  4. Tentukan tanggal mulai & akhir (auto-suggest per 2 minggu)
  5. Masukkan nominal anggaran (format Rp otomatis)

Monitoring Anggaran

  3. Isi form: Kategori, Jumlah (Rp), Deskripsi
  4. Upload lampiran (opsional)
  5. Klik Simpan
  6. Klik Detail untuk melihat lampiran yang sudah diupload

8. Pembukuan

Modul Pembukuan mencatat seluruh transaksi keuangan dalam bentuk jurnal akuntansi dan buku besar.

8.1 Jurnal Umum

Jurnal otomatis dibuat dari transaksi: distribusi, konsumsi bahan, dan waste. Anda juga bisa menambah jurnal manual.

  1. Buka Pembukuan → Jurnal Umum
  2. Lihat daftar jurnal yang sudah tercatat
  3. Untuk menambah jurnal manual, klik + Tambah Jurnal
  4. Isi tanggal, akun debit, akun kredit, nominal, dan keterangan
  5. Klik Simpan
  6. Gunakan filter periode untuk melihat jurnal per bulan/tahun

8.2 General Ledger (Buku Besar)

Buku besar menampilkan ringkasan transaksi per akun COA (Chart of Accounts).

  1. Buka Pembukuan → General Ledger
  2. Pilih akun COA yang ingin dilihat
  3. Filter per periode (bulan/tahun)
  4. Lihat saldo awal, mutasi debit/kredit, dan saldo akhir

8.3 Tutup Buku

Tutup periode akuntansi untuk mengunci jurnal pada periode tersebut.

  1. Buka Pembukuan → Tutup Buku
  2. Pilih periode yang akan ditutup
  3. Klik Tutup Periode
  4. Jurnal pada periode tersebut tidak bisa diedit lagi
🔴 Penting: Tutup buku bersifat permanen. Pastikan semua jurnal sudah benar sebelum menutup periode.
Manual Book ERP MBG v1.1.0 — PT. Manggala Utama Indonesia

11. Notifikasi & Chat

11.1 Notifikasi

Sistem mengirim notifikasi real-time untuk setiap aktivitas penting.

  1. Klik ikon 🔔 di header atas
  2. Badge merah menunjukkan jumlah notifikasi belum dibaca
  3. Klik notifikasi untuk membuka halaman terkait

Notifikasi Otomatis Dikirim Untuk:

  • IR diajukan → notifikasi ke approver
  • IR di-approve → notifikasi ke peminta + DO dibuat
  • DO dikirim → notifikasi ke admin dapur
  • KR selesai → notifikasi ke admin gudang
  • PO diajukan → notifikasi ke approver
  • Pembayaran diajukan → notifikasi ke finance

11.2 Chat

Fitur chat internal untuk komunikasi antar pengguna sistem.

  1. Klik ikon 💬 di header atas
  2. Pilih kontak dari daftar pengguna
  3. Ketik pesan dan tekan Enter atau klik kirim
  4. Tanda centang: ✓ terkirim, ✓✓ dibaca
  5. Badge unread count muncul per kontak
💡 Tip: Chat panel bisa diminimize agar tidak mengganggu pekerjaan. Notifikasi pesan baru tetap muncul di badge.
Manual Book ERP MBG v1.1.0 — PT. Manggala Utama Indonesia

12. Telegram Bot

12.1 Menghubungkan Akun

  2. Ketik /start
  3. Bot meminta email → masukkan email yang terdaftar di sistem
  4. Akun terhubung ✅

12.2 Upload IR via Telegram

  1. Kirim file Excel (template SPPG) ke bot

12.3 Approve via Telegram

  2. Klik tombol Approve di pesan Telegram
  3. IR/PO langsung ter-approve di sistem

13. Pengaturan

Menu Pengaturan hanya tersedia untuk role Owner dan Super Admin (kecuali Profil Saya yang tersedia untuk semua role).

13.1 Admin Panel

Panel administrasi menampilkan overview statistik sistem dan akses ke pengaturan lanjutan.

  • Overview: Statistik pengguna, transaksi, dan sistem
  • Users: Kelola pengguna (lihat 13.2)
  • Settings: Pengaturan sistem global
  • Audit: Log aktivitas (lihat 13.3)
  • Announcements: Buat pengumuman untuk semua pengguna

13.2 Kelola Pengguna

  1. Buka Pengaturan → Pengguna & Akses
  2. Tambah User: Klik + → isi nama, email, password, role, dapur (jika Admin Dapur)
  3. Edit User: Klik Edit → ubah nama, email, role, dapur
  4. Reset Password: Klik Reset → masukkan password baru
💡 Tip: Saat menambah user dengan role Admin Dapur, wajib memilih dapur yang akan dikelola.

13.3 Audit Log

Audit Log mencatat seluruh aktivitas pengguna di sistem untuk keperluan tracking dan keamanan.

  1. Buka Pengaturan → Audit Log
  2. Lihat semua aktivitas: siapa, kapan, aksi apa, pada data apa
  3. Filter per user, tanggal, dan tipe aksi

13.4 Profil Saya

  1. Buka Pengaturan → Profil Saya
  2. Edit nama dan email
  3. Ganti password (masukkan password lama dan password baru)
  4. Klik Simpan

16. FAQ & Troubleshooting

Pertanyaan yang sering diajukan dan solusi untuk masalah umum.

Login & Akses

Q: Tidak bisa login?

A: Pastikan email dan password benar. Perhatikan huruf besar/kecil pada password. Jika lupa password, minta Super Admin untuk reset melalui menu Pengaturan → Pengguna.

Q: Menu tertentu tidak muncul di sidebar?

A: Menu yang tampil disesuaikan dengan role Anda. Admin Dapur hanya melihat menu yang relevan dengan operasional dapur. Hubungi Super Admin jika merasa ada menu yang seharusnya tersedia.

Data & Transaksi

Q: Data tidak muncul atau kosong?

A: Cek role Anda — Admin Dapur hanya melihat data dapurnya sendiri. Coba refresh halaman (Ctrl+Shift+R). Pastikan filter periode sudah benar.

Q: Invoice tidak muncul setelah Kitchen Receiving?

A: Invoice otomatis dibuat saat Kitchen Receiving dikonfirmasi. Pastikan KR sudah di-confirm (klik Konfirmasi Penerimaan), bukan hanya dilihat.

Q: Budget warning tidak muncul saat buat IR?

A: Pastikan sudah ada anggaran aktif untuk dapur tersebut di periode saat ini. Buat anggaran terlebih dahulu di menu Arus Kas → Anggaran Dapur.

Upload & File

Q: Upload Excel gagal?

A: Pastikan format file .xlsx (bukan .xls). Gunakan template SPPG yang sesuai. Pastikan ada kolom: No, Bahan, Qty, Satuan. Jangan mengubah struktur header template.

Q: Upload bukti pembayaran gagal?

A: Format yang didukung: JPG, PNG, dan PDF. Pastikan ukuran file tidak melebihi batas (maks 5MB). Coba kompres file jika terlalu besar.

Telegram Bot

Q: Telegram bot tidak merespons?

A: Pastikan sudah mengetik /start dan menghubungkan email. Cek apakah email yang dimasukkan sama persis dengan yang terdaftar di sistem (termasuk huruf besar/kecil).

Q: Notifikasi Telegram tidak masuk?

A: Pastikan akun Telegram sudah terhubung. Cek apakah bot tidak di-block. Pastikan notifikasi Telegram tidak di-mute.

Cetak & PDF

Q: Bagaimana cara cetak/save PDF?

A: Klik tombol PDF atau Cetak pada halaman terkait. Browser akan membuka dialog print. Pilih "Save as PDF" sebagai printer, atau langsung print ke printer fisik.

Q: Harga tidak muncul separator titik?

A: Input harga menggunakan komponen khusus. Ketik angka saja (tanpa titik atau koma), separator titik ribuan akan otomatis muncul.
